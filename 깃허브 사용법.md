# 깃허브 기본 사용법 요약 (**~**, **`~`**표시는 생략, ()표시도 생략)

### 1. 새로운 브랜치 만들고 작업 올리기
나만의 작업 공간(브랜치)을 만들고 깃허브에 처음 올릴 때 사용하는 순서입니다.

1. 새 브랜치를 만들고 바로 그곳으로 이동합니다.
   **`git checkout -b (새 브랜치 이름)`**
2. 방금 만든 새 브랜치를 깃허브 서버에도 등록합니다. (새 브랜치 생성 후 처음 한 번만 입력)
   **`git push -u origin (새 브랜치 이름)`**      
3. 수정한 파일들을 저장할 준비를 합니다. (점 `.` 앞에 띄어쓰기 필수)
   **`git add .`**
4. 어떤 작업을 했는지 **`메모`**와 함께 컴퓨터에 저장(커밋)합니다.
   **`git commit -m "메모"`**
5. 저장된 내용을 깃허브로 현재 브랜치에 최종 업로드(푸시)합니다.
   **`git push origin (새 브랜치 이름)`**
      
**Ex. '현진' 브랜치 생성 후 작업 올리기**
   **`git checkout -b 현진`**
   **`git push -u origin 현진`**
   **`git add .`**
   **`git commit -m "메모"`**
   **`git push origin 현진`**

**Ex. 기존 'main' 브랜치에 작업 올리기**
   1) 파일 수정하기
   2) 파일 확실히 저장하기 (Ctrl + S / Mac은 Cmd + S)
   3) Git 명령어 순서대로 입력하기
      **`git add .`**
      **`git commit -m "메모"`**
      **`git push origin main`**

---

### 2. 브랜치 업데이트 및 이동

1. 현재 브랜치의 최신 상태를 다운로드(업데이트)할 때
   **`git pull origin (현재 브랜치 이름)`**      
2. 다른 브랜치로 화면(작업 공간)을 전환하고 싶을 때
   **`git checkout (이동할 브랜치 이름)`**
      
**Ex. '현진' 브랜치에서 '주형' 브랜치로 전환**
   **`ex. git pull origin 현진`**
   **`ex. git checkout 주형`**
   
---

### 3. 기존 파일 삭제하고 깃허브에 반영하기
1. 에디터에서 파일 지우기: Cursor 에디터 왼쪽 파일 목록에서 지우고 싶은 파일을 마우스 우클릭한 뒤, **Delete(삭제)**를 눌러서 지워줍니다.
2. Git 명령어 순서대로 입력: 터미널에 늘 쓰시던 3단 콤보를 똑같이 입력해 줍니다.
   **`git add .`**
   **`git commit -m "삭제 이유"`**
   **`git push origin (현재 브랜치 이름)`**

**Ex. '현진' 브랜치에서 파일 삭제하고 반영하기**
   1) 에디터에서 파일 지우기
   2) Git 명령어 순서대로 입력
      **`git add .`**
      **`git commit -m "삭제 이유"`**
      **`git push origin 현진`**

### 4. 긴급 복구 명령어 (주의)
실수로 지운 파일 원래대로 복구하기

   **`git restore (파일 이름)`**
